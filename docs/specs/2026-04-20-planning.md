# 보험 청구 도우미 서비스 기획 문서

작성일: 2026-04-20

---

## 1. 서비스 개요

### 목적
보험 약관 PDF를 업로드하면 AI가 약관을 분석하여, 가입자가 처한 상황에서 보험 청구가 가능한지 사전에 판단하고 약관 조항을 가입자 편에서 해석해주는 서비스.

### 핵심 가치 제안
- 어려운 보험 약관을 일반인이 쉽게 이해할 수 있도록 해석
- 치료 전/후 청구 가능 여부를 미리 확인하여 불필요한 분쟁 예방
- 약관 원문 근거를 직접 제시하여 신뢰성 확보

---

## 2. 사용자 시나리오 (User Story)

### 비로그인 사용자
1. 서비스에 접속
2. 보험 약관 PDF 업로드
3. 챗봇에 상황 설명 (예: "허리 디스크로 MRI 찍었는데 청구 가능한가요?")
4. AI가 필요한 경우 추가 정보 질문 (나이, 성별, 가입 기간 등)
5. AI가 청구 가능 여부 + 근거 조항 하이라이트로 답변
6. 세션 내에서만 대화 이력 유지 (브라우저 닫으면 소멸)

### 로그인 사용자
1. 위 비로그인 시나리오와 동일하게 사용
2. 추가로 분석 이력이 계정에 저장됨
3. 다음 방문 시 이전 분석 이어서 확인 가능
4. 한 번 입력한 개인 정보(나이, 성별 등)는 다음 분석 시 재활용

---

## 3. 핵심 기능 목록

### PDF 업로드 및 파싱
- 사용자가 보험 약관 PDF를 직접 업로드
- 서버에서 PDF 텍스트 추출 및 파싱
- 파싱된 내용을 Claude API 컨텍스트로 활용

### 대화형 챗봇 (Claude API 연동)
- 사용자가 자유롭게 상황을 설명하면 AI가 대화 형식으로 답변
- 답변에 필요한 정보(나이, 성별, 상병명 등)는 AI가 대화 중 자연스럽게 질문
- 약관 근거 조항을 함께 제시

### 약관 근거 조항 표시
- AI 답변 시 참조한 약관 원문 조항을 함께 표시
- 가입자가 직접 원문을 확인할 수 있도록 제공

### 회원 기능
- 이메일 로그인 (Google)
- 비로그인으로도 핵심 기능 사용 가능
- 로그인 시 분석 이력 저장 및 재열람

### 분석 이력 관리
- 로그인 사용자의 과거 채팅 세션 목록 조회
- 이전 세션 이어보기

## 4. 비기능 요구사항

### 파일 제한
- PDF 파일 크기 제한 설정 필요 (예: 최대 20MB)
- PDF 형식만 허용

### 기술 스택
- 백엔드: Java 21, Spring Boot
- 프론트엔드: React (최대한 단순하게 구성)
- AI (생성): Claude API (Anthropic) — 약관 해석 및 답변 생성
- AI (검색/RAG): Spring AI — OpenAI Embedding 연동, pgVector VectorStore, RAG 파이프라인 통합 관리
- DB: PostgreSQL 16 + pgVector (벡터 저장 및 유사도 검색을 RDB 하나로 통합)
- DB 접근: Spring Data JPA (Hibernate) — 일반 엔티티 CRUD / 벡터 검색은 Spring AI VectorStore로 처리

### RAG 처리 흐름
1. PDF 업로드 → 텍스트 추출 → chunk 분할
2. 각 chunk를 OpenAI Embedding API로 벡터 변환 → pgVector에 저장
3. 사용자 질문 입력 → 질문도 벡터 변환
4. pgVector 유사도 검색으로 관련 조항만 추출
5. 추출된 조항 + 질문을 Claude API에 전달 → 답변 생성
- 이유: 약관 PDF 전체를 매번 Claude API에 전달하면 토큰 비용이 과다하게 발생하므로, 관련 조항만 선별하여 비용 절감 및 정확도 향상
