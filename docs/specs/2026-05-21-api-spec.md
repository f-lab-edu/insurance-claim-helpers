# REST API 엔드포인트 스펙

작성일: 2026-05-21

---

## 공통 사항

### Base URL

```
/api
```

### 인증 방식

- Spring Security OAuth2 + Spring Session 기반 쿠키 인증 (`JSESSIONID`)
- 로그인 사용자: Google OAuth 완료 후 Spring Security가 세션에 인증 정보 저장
- 비로그인 사용자: HTTP 세션 ID(`JSESSIONID`)를 `session_key`로 사용
- 현재 세션 저장소: 메모리(`MapSessionRepository`) — 추후 JDBC/Redis로 교체 가능

### 에러 응답 형식

모든 에러는 아래 형식으로 반환한다.

```json
{
  "code": "POLICY_NOT_FOUND",
  "message": "해당 약관을 찾을 수 없습니다."
}
```

| HTTP 상태 | 의미 |
|-----------|------|
| 400 | 요청 파라미터 오류 |
| 401 | 인증 필요 (로그인 사용자 전용 API) |
| 403 | 접근 권한 없음 (다른 사용자 리소스) |
| 404 | 리소스 없음 |
| 500 | 서버 내부 오류 |

## 1. Auth

### 1.1 Google OAuth 로그인 시작

```
GET /auth/google
```

Google OAuth 인증 페이지로 리다이렉트한다.

**응답**

| 상태 | 설명 |
|------|------|
| 302 | Google 로그인 페이지로 리다이렉트 |

---

### 1.2 Google OAuth 콜백

```
GET /auth/google/callback
```

Google에서 인증 완료 후 호출되는 콜백 엔드포인트. 서버는 사용자 조회 또는 신규 생성 후 세션에 저장한다.

**쿼리 파라미터**

| 이름 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `code` | string | O | Google 인가 코드 |
| `state` | string | - | CSRF 방지용 |

**응답**

| 상태 | 설명 |
|------|------|
| 302 | 로그인 전 페이지 또는 `/` 로 리다이렉트 |

---

### 1.3 로그아웃

```
POST /auth/logout
```

현재 세션을 무효화한다.

**응답**

| 상태 | 설명 |
|------|------|
| 204 | 로그아웃 성공, 본문 없음 |

## 2. Policy (약관 PDF)

### 2.1 PDF 업로드

```
POST /api/policies
```

보험 약관 PDF를 업로드한다. 업로드 즉시 백그라운드에서 텍스트 추출 → 벡터 저장이 진행된다.

**요청**

- Content-Type: `multipart/form-data`

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `file` | File | O | PDF 파일 (최대 20MB) |

**응답 (201 Created)**

```json
{
  "id": "3f7a1b2c-...",
  "fileName": "삼성생명_종신보험.pdf",
  "fileSize": 1048576,
  "status": "PROCESSING",
  "createdAt": "2026-05-21T10:00:00"
}
```

> `status`는 `PENDING → PROCESSING → COMPLETED / FAILED` 순서로 변경된다.
> 클라이언트는 `status`가 `COMPLETED`가 될 때까지 폴링하거나, 추후 웹소켓/SSE로 개선 가능.

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `INVALID_FILE_TYPE` | 400 | PDF가 아닌 파일 |
| `FILE_SIZE_EXCEEDED` | 400 | 20MB 초과 |

---

### 2.2 약관 처리 상태 조회

```
GET /api/policies/{policyId}/status
```

업로드된 약관의 처리 상태를 조회한다. PDF 업로드 직후 폴링에 사용한다.

**경로 파라미터**

| 이름 | 설명 |
|------|------|
| `policyId` | 약관 UUID |

**응답 (200 OK)**

```json
{
  "id": "3f7a1b2c-...",
  "status": "COMPLETED"
}
```

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `POLICY_NOT_FOUND` | 404 | 해당 약관 없음 |
| `FORBIDDEN` | 403 | 다른 사용자/세션의 약관 |

---

## 3. Chat (채팅)

### 3.1 채팅 세션 생성

```
POST /api/chat/sessions
```

새로운 채팅 세션을 생성한다. "지금 시작하기" 또는 첫 PDF 첨부 시 호출한다.

**요청 본문**

없음

**응답 (201 Created)**

```json
{
  "id": "a1b2c3d4-...",
  "policies": [],
  "createdAt": "2026-05-21T10:00:00"
}
```

---

### 3.2 채팅 세션 목록 조회

```
GET /api/chat/sessions
```

로그인 사용자의 채팅 세션 목록을 반환한다. 마이페이지 "최근 상담 내역"에 사용한다.

**응답 (200 OK)**

```json
[
  {
    "id": "a1b2c3d4-...",
    "policies": [
      { "id": "3f7a1b2c-...", "fileName": "삼성생명_종신보험.pdf" }
    ],
    "lastMessage": "해당 약관 3조 2항에 따르면...",
    "createdAt": "2026-05-21T10:00:00"
  }
]
```

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `UNAUTHORIZED` | 401 | 비로그인 접근 |

---

### 3.3 채팅 세션 메시지 이력 조회

```
GET /api/chat/sessions/{sessionId}
```

특정 세션의 연결된 약관과 전체 메시지 이력을 반환한다.

**경로 파라미터**

| 이름 | 설명 |
|------|------|
| `sessionId` | 채팅 세션 UUID |

**응답 (200 OK)**

```json
{
  "id": "a1b2c3d4-...",
  "policies": [
    { "id": "3f7a1b2c-...", "fileName": "삼성생명_종신보험.pdf", "status": "COMPLETED" }
  ],
  "messages": [
    {
      "id": "m1-...",
      "role": "USER",
      "content": "암 진단비 청구 가능한가요?",
      "createdAt": "2026-05-21T10:01:00"
    },
    {
      "id": "m2-...",
      "role": "ASSISTANT",
      "content": "해당 약관 3조 2항에 따르면...",
      "createdAt": "2026-05-21T10:01:05"
    }
  ]
}
```

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `SESSION_NOT_FOUND` | 404 | 해당 세션 없음 |
| `FORBIDDEN` | 403 | 다른 사용자/세션의 채팅 |

---

### 3.4 메시지 전송 (SSE 스트리밍)

```
POST /api/chat/sessions/{sessionId}/messages
```

사용자 메시지를 전송하고 AI 답변을 SSE로 스트리밍한다.
사용자 메시지 저장 → RAG로 관련 조항 검색 → Claude API 스트리밍 응답 → 완료 시 DB 저장 순서로 처리한다.

**경로 파라미터**

| 이름 | 설명 |
|------|------|
| `sessionId` | 채팅 세션 UUID |

**요청 본문**

```json
{
  "content": "암 진단비 청구 가능한가요?"
}
```

**요청 헤더**

```
Accept: text/event-stream
```

**응답 (200 OK)**

- Content-Type: `text/event-stream`

```
data: {"delta": "해당 약관 "}

data: {"delta": "3조 2항에 따르면 "}

data: {"delta": "암 진단 시 청구 가능합니다."}

data: {"done": true, "messageId": "m2-uuid-..."}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `delta` | string | 스트리밍 텍스트 조각 |
| `done` | boolean | 스트림 완료 여부 |
| `messageId` | string | 완료 시 저장된 AI 메시지 UUID |

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `SESSION_NOT_FOUND` | 404 | 해당 세션 없음 |
| `FORBIDDEN` | 403 | 다른 사용자/세션의 채팅 |
| `NO_POLICY_ATTACHED` | 400 | 세션에 약관이 연결되지 않음 |

---

### 3.5 세션에 약관 연결

```
POST /api/chat/sessions/{sessionId}/policies
```

채팅 세션에 약관을 추가한다. 채팅창에서 PDF를 첨부하거나 추가 시 호출한다.

**경로 파라미터**

| 이름 | 설명 |
|------|------|
| `sessionId` | 채팅 세션 UUID |

**요청 본문**

```json
{
  "policyId": "3f7a1b2c-..."
}
```

**응답 (200 OK)**

```json
{
  "chatSessionId": "a1b2c3d4-...",
  "policyId": "3f7a1b2c-...",
  "fileName": "삼성생명_종신보험.pdf"
}
```

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `SESSION_NOT_FOUND` | 404 | 해당 세션 없음 |
| `POLICY_NOT_FOUND` | 404 | 해당 약관 없음 |
| `POLICY_NOT_COMPLETED` | 400 | 약관이 아직 처리 중 (`COMPLETED`가 아님) |
| `FORBIDDEN` | 403 | 다른 사용자/세션의 리소스 |

---

### 3.6 세션에서 약관 연결 해제

```
DELETE /api/chat/sessions/{sessionId}/policies/{policyId}
```

채팅 세션에서 약관을 제거한다. 채팅창에서 PDF 칩의 ✕ 버튼 클릭 시 호출한다.

**경로 파라미터**

| 이름 | 설명 |
|------|------|
| `sessionId` | 채팅 세션 UUID |
| `policyId` | 약관 UUID |

**응답**

| 상태 | 설명 |
|------|------|
| 204 | 연결 해제 성공, 본문 없음 |

| 에러 코드 | HTTP | 설명 |
|-----------|------|------|
| `SESSION_NOT_FOUND` | 404 | 해당 세션 없음 |
| `POLICY_NOT_FOUND` | 404 | 해당 약관 없음 |
| `FORBIDDEN` | 403 | 다른 사용자/세션의 리소스 |
